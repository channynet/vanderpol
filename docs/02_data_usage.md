# Data Usage

## Short Answer

The `v001_full_pipeline` result uses:

- `1` synthetic simulator-generated treatment dataset
- `1` external ECG dataset for noise realism

The treatment success/failure labels are not taken from real patients. They are
generated by the simulator.

## Data Types

| Data source | Used in v001? | Role |
| --- | --- | --- |
| Simulator-generated episodes | Yes | Treatment outcome, reward, selector training/evaluation. |
| PhysioNet/CinC Challenge 2015 | Yes | Noise/OOD profile estimation and robustness tests. |
| MIT-BIH Arrhythmia Database | Not directly in v001 figures | Real abnormal ECG morphology validation; used in earlier abnormal-window extraction. |
| CUDB | Not directly in v001 figures | VT/VF waveform validation target for future versions. |
| PTB-XL | Not yet | Optional future encoder pretraining. |
| AHA/ICD/resonant-drift literature | Yes, as config anchors | Calibration ranges and rule-baseline justification. |

## Important Distinction

External ECG data is used for observation realism and validation.

It is not used as treatment-outcome data because MIT-BIH, CUDB, and Challenge
2015 do not provide labels like:

```text
ATP succeeded
cardioversion failed
defibrillation restored rhythm
```

Therefore:

| Task | Data |
| --- | --- |
| ECG morphology/noise realism | Real ECG datasets |
| Treatment success/failure | Simulator episodes |
| Selector learning | Simulator outcome matrix |
| Calibration and clinical thresholds | Guideline/literature anchors |

## Current v001 Scale

| Quantity | Value |
| --- | ---: |
| Rhythm scenarios | 5 |
| Patients per scenario | 20 |
| Patient contexts | 100 |
| Algorithms per patient | 5 |
| Base treatment episodes | 500 |
| Selector train contexts | 70 |
| Selector eval contexts | 30 |

## Scenario Scope

The current scenario set is intentionally kept at five rhythms:

- NSR
- SVT/flutter-like
- monomorphic VT
- polymorphic VT
- VF-like chaos

MIT-BIH labels such as AFIB, ventricular bigeminy, ventricular trigeminy, IVR,
and nodal rhythm are useful external reference data, but they are not being
added as simulator scenarios in the next iteration.

## Main Data Limitation

The current synthetic abnormal ECGs are not yet sufficiently matched to real
MIT-BIH/CUDB abnormal rhythms. This limits how strongly the selector result can
be interpreted.
